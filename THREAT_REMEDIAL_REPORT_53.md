## 2. 実装詳細

### 2.1 真の Peer Identity に基づく制限 (F-52-1)
これまでの実装ではセッションごとの一時鍵（ephemeral）を PeerID の計算に使っていたため、攻撃者が毎回新しい一時鍵を作るだけで制限を回避できていました。
- **指紋計算の修正**: ハンドシェイクで提示され、検証に成功した「長期署名公開鍵」をハッシュ化して PeerID とするように変更しました。これにより、同一の鍵を持つユーザーは再接続時にも確実に同一人物として識別され、クールダウンが適用されます。

### 2.2 防御ロジックの効率化と網羅 (F-52-3, F-52-4)
- **リソース保護**: 接続を受け入れた直後、タスクを `spawn` する前に IP ベースの超短期（2秒）クールダウンを判定。無効な接続のためにメモリや CPU を消費することを防ぎます。
- **検証失敗への対処**: 正しいハンドシェイクを装いつつ署名検証で失敗させる攻撃に対し、IP ベースのペナルティを課すようにしました。

### 2.3 マルチバックエンドの明示的制限 (F-51-2)
OpenSSL バックエンドでは PQC 固有の OID を含む PKCS#8 の復号が標準では困難です。
- **フェイルファスト**: OpenSSL バックエンドで暗号化 PQC 鍵が検出された場合、サイレントに失敗するのではなく、RustCrypto バックエンドの使用を推奨する具体的なエラーを表示するようにしました。
