# SECURITY_PROFILE: GUI Layer

GUI 層 (M1〜M5) で導入された防護機構と固有脅威の整理。

## 1.1 M1: パスフレーズ入力
- **セキュアフィールド**: `input-type: password` を採用し、入力中の平文露出を防止。
- **メモリ管理**: 入力後の即時消去（`set_text("")`）および `Zeroizing` によるメモリ上での機密保護。

## 1.3 M3: クリップボード保護
- **自動消去**: 接続チケット等の機密情報をコピーした場合、30 秒後にクリップボードを自動クリア。

## 1.5 M2: QR コードスキャン入力の脅威モデル

- **カメラ権限の最小化**: 常時 ON を禁止し、ユーザーが「Scan QR」ボタンを明示的に押下した期間のみデバイスをオープンする。
- **機密情報の非露出**: スキャンした QR ペイロードおよびカメラフレームをログ (stdout/stderr/file) に出力しない。
- **入力検証の強制**: `Ticket::from_str` によるパースに成功した文字列のみを UI 入力欄に注入する。

## 1.6 M4: デスクトップ通知の脅威モデル

- **本文非機密化ポリシー**: 通知本文に受信メッセージ内容を含めず、`<peer-id> から新しいメッセージがあります` の汎用文字列のみを表示する。これによりロック画面・通知センター履歴経由の漏洩を構造的に防止。
- **Leading-edge レート制限**: バースト的な受信に対し、最初の 1 件は即時通知、5 秒間は後続抑制。妨害を最小化しつつ即時性を保つ。
- **フォーカス抑制**: アプリがアクティブな間は通知を発火しない。
- **クリック挙動**: 通知クリックでアプリウィンドウを前面に raise (Linux Wayland では Best effort)。

## 1.7 M5: スクリーンショット盗み見対策

- **オプトイン防護**: ユーザ設定（Privacy Mode）により、OS のキャプチャ防止 API を有効化。
  - Windows: `SetWindowDisplayAffinity` (WDA_EXCLUDEFROMCAPTURE)
  - macOS: `NSWindow.sharingType = .none`
- **限界の明記**: 本機能は OS レベルの標準的なキャプチャ（スクショ、録画、画面共有）を制限するものであり、物理的なカメラ撮影や低レイヤの不正プログラムによる取得を完全に防ぐものではない。
- **プラットフォーム制約**: Linux (X11/Wayland) 環境では OS/Compositor 側のセキュリティプロトコル（xdg-desktop-portal 等）に依存し、本アプリからの強制的な制限は行わない。
