# Iroh 移行プロジェクト 最終完了レポート (nkCryptoTool-rust)

作成日: 2026-05-06
ステータス: 完了 (全 4 フェーズ完了)

## 1. プロジェクト概要

`nkCryptoTool-rust` のネットワークモード（チャット・ファイル転送）を、従来のレガシーな TCP 直接接続から、現代的な P2P トランスポートライブラリである [Iroh](https://iroh.computer/) へ移行しました。

### 主な目的
- **NAT 越えの自動化**: ポート開放なしでモバイル回線や NAT 背後からの接続を可能にする。
- **セキュリティの強化**: NodeId ベースのアドレッシングと、PQC（耐量子暗号）による二重保護の実現。
- **利便性の向上**: 接続情報を集約した Ticket 形式と QR コードによる簡便なピアリング。

---

## 2. 実施内容と成果

### フェーズ 1: 基盤構築とセキュリティ強化
- **モジュール刷新**: `src/network/` ディレクトリにトランスポート非依存の共通ロジックを抽出し、Iroh 向けの `iroh.rs` を新規実装しました。
- **PQC 認証の移植**: ML-KEM + ML-DSA + ECC ハイブリッドハンドシェイクを Iroh ストリーム上で再実装。
- **チャネルバインディング**: PQC ハンドシェイクに Iroh の NodeId を組み込み、トランスポート層と暗号層を強固に紐付けました。

### フェーズ 2: Ticket フォーマットと MITM 防御
- **NKCT1 Ticket**: NodeId、リレー URL、接続アドレス、PQC 鍵指紋、チェックサムを内包したバイナリベースの Ticket 形式を実装。
- **MITM 検知**: Ticket に含まれる指紋と実際の公開鍵を照合し、中間者攻撃を確実に遮断する機能を実装。
- **QR コード対応**: 接続用 Ticket を ASCII QR コードとして表示する機能を実装。

### フェーズ 3: プロトコル拡張とファイル転送
- **V3.1 ハンドシェイク**: サーバーの KEM 公開鍵も送信するように拡張し、暗号化鍵の指紋検証（MITM 対策）を完全化。
- **ALPN 分離**: チャットとファイル転送を ALPN レベルで分離し、1 ポートでのマルチプロトコル共存を実現。
- **マルチクライアント認証**: クライアントの DSA 公開鍵を送信する拡張により、特定の鍵ファイルを置かずに Allowlist だけで不特定多数の認証済みピアを受け入れ可能にしました。

### フェーズ 4: 非推奨化とドキュメント整備
- **TCP の廃止予定化**: レガシー TCP モード使用時に警告を表示し、将来の削除に向けた移行期間を開始。
- **仕様ドキュメント更新**: `SPEC.md` および `README.md` に Iroh プロトコルと Ticket 形式の詳細を追記。

## 3. 検証ステータス

`src/network/iroh.rs` 内の E2E テストスイート（全 8 ケース）により、以下の正常性と安全性を実証済みです。
- **正常系**: 未認証接続、署名認証接続、マルチクライアント認証、チャット/ファイル転送の初期化。
- **異常系（負例）**: 署名不正の検知、指紋不一致（MITM）の検知、Allowlist 外の拒否。

全てのテストが `serial_test` により安定して 1 秒以内に完了します。

## 4. 今後の課題

- **フォワードセクレシ (Forward Secrecy)**: 現在の PQC ハンドシェイクは静的長期鍵ベースです。将来的にエフェメラル KEM 鍵 + Ratchet プロトコルの導入を検討します。
- **ファイルパス指定の転送**: 現在の Iroh ファイル転送は stdin/stdout パイプ専用です。CLI レベルでのファイルパス直接指定への対応を予定しています。
