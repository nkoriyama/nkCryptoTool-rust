# 修正レポート — F-CLI-FORCE: `--force` フラグの修正

作成日: 2026-05-08
ステータス: 修正完了 (Verified)

## 概要

`--force` CLI フラグが、暗号化（`--encrypt`）および復号（`--decrypt`）の出力ファイル作成時に無視され、既存ファイルの上書きができない不具合（F-CLI-FORCE）を修正しました。

## 根本原因

`src/processor.rs` 内の `run_streaming_encrypt` および `run_streaming_decrypt` 関数において、出力ファイルを開く際に `config.force` フラグが参照されておらず、常に `create_new(true)` (O_EXCL) が使用されていたため、ファイルが存在すると必ずエラーとなっていました。

## 修正内容

### 1. `run_streaming_encrypt` の修正
- 引数に `force: bool` を追加しました。
- `force` が `true` の場合、ファイルを開く前に `std::fs::remove_file` を試行するようにしました。
- これにより、`create_new(true)` と `O_NOFOLLOW` によるセキュリティ特性（シンボリックリンク攻撃の防御）を維持したまま、安全な上書きが可能になりました。

### 2. `run_streaming_decrypt` の修正
- 引数に `force: bool` を追加しました。
- 復号処理完了後のファイル名変更（rename）時に、`force` が `false` かつ出力先が既に存在する場合にエラーを返すようにロジックを厳密化しました。
- `force` が `true` の場合は、従来通り `rename` による上書きを許容します。

### 3. `encrypt_file` / `decrypt_file` の更新
- 上記ストリーミング関数への委譲時に `config.force` を正しく渡すように修正しました。

## 検証結果

自作の再現スクリプトを用いて、以下のケースが正常に動作することを確認しました。

| ケース | 期待挙動 | 結果 |
|---|---|---|
| 初回暗号化 (新規作成) | 成功 | ✅ 成功 |
| `--force` なしで既存ファイルへ暗号化 | エラー (File exists) | ✅ 成功 (期待通り失敗) |
| `--force` ありで既存ファイルへ暗号化 | 上書き成功 | ✅ 成功 |
| `--force` なしで既存ファイルへ復号 | エラー (File exists) | ✅ 成功 (期待通り失敗) |
| `--force` ありで既存ファイルへ復号 | 上書き成功 | ✅ 成功 |

## 影響範囲
- CLI スクリプトやベンチマークなど、同じ出力先を再利用する自動化処理が正しく動作するようになりました。
- セキュリティ上の `O_NOFOLLOW` 保護は維持されています。
