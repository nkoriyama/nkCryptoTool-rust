## 指摘事項への対応 (v3 再検証に基づく)

### 1. 認証ログの正確化 (F-IROH-22)
公開鍵による署名検証が実際に行われ、成功したコードパス内でのみ「Authenticated successfully」と表示するようにログ出力を厳格化しました。これにより、未認証許可モード実行時のログの混乱を解消しました。

### 2. Allowlist ロジックのデッドコード削除と厳格化 (F-IROH-16)
デッドコードとなっていた条件分岐を整理し、Allowlist が有効な場合は匿名ピア（署名なしピア）を即座に拒否するロジックを確実に機能するように修正しました。

### 3. Ticket フォーマットの正規化と互換性確保 (F-IROH-19)
- `nkct1` プレフィックスを復活させ、Base32 エンコードされたバイナリペイロード（`iroh::NodeAddr` のシリアライズ版）を採用しました。
- これにより、NodeId だけでなく直接接続アドレスも同梱され、接続の堅牢性が向上しました。

### 4. リソース管理の Drop ガード化 (F-IROH-25)
`EndpointGuard` (RAII) を導入し、エラーによる早期 return やパニック発生時でも、Iroh Endpoint が確実に `close` されるようにしました。これによりバックグラウンドタスクのリークやテストのハングを根本的に防止しました。

### 5. 負例テストによるセキュリティ境界の検証 (F-IROH-12 拡充)
「署名不正」「Allowlist による拒否」「指紋不一致（MITM）」といった負例テストを 3 件追加し、正常系以外の接続が期待通りに遮断されることを実証しました。
