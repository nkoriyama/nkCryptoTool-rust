## 2. 実装詳細

### 2.1 秘密鍵復号ロジックの厳格化 (F-50-1, F-51-2)
`extract_raw_private_key` の信頼性を「機能を装ったサイレント失敗」がないレベルまで引き上げました。
- **エラー伝搬の徹底**: 鍵が暗号化されている場合、復号に成功しない限り値を返さない構造にしました。これにより、間違ったパスフレーズによる不透明な二次エラーを排除しました。
- **バックエンドごとの明示性**: OpenSSL バックエンドでは PQC 鍵の暗号化解除が困難なため、該当する場合はユーザーに RustCrypto バックエンドの使用を促す明確なメッセージを返すようにしました。

### 2.2 Peer 識別ベースの DoS 防御 (F-51-1, F-50-3)
IP ベースの制限が持つ「NAT 越しの正規ユーザーを巻き添えにする」という本質的課題を、認証情報を活用することで解決しました。
- **PeerID の導入**: 接続ピアを「公開鍵の指紋」または「IP（未認証時）」で識別するように変更。これにより、同じ NAT 共有 IP からでも、異なる正当な公開鍵を持つユーザーは個別にセッションを確立できるようになりました。
- **一律のクールダウン**: 攻撃者によるセッション時間の調整（31秒維持など）を無効化するため、セッションの長短に関わらず、切断後には必ず保護期間（クールダウン）を設けるようにしました。

### 2.3 信頼性の高い非同期実装 (F-50-2)
- **標準的 Mutex 採用**: `parking_lot::Mutex` を全面的に採用。`Drop` 内での同期的なロック取得を安全に行えるようにし、Mutex poisoning による連鎖パニックのリスクを完全に解消しました。

## 4. 総括と今後の展望

本修正サイクル（v44〜52）を通じて、`nkCryptoTool-rust` は PQC（ポスト量子暗号）実装特有の複雑な鍵管理と、ネットワークモードにおける DoS 攻撃耐性の両面で、極めて堅牢な基礎を構築しました。

- **残された中長期課題**:
    - **ASN.1 完全構造パース**: 現在のヒューリスティック走査（信頼性は大幅に向上済み）を、`pkcs8` クレート等を用いた完全な構造解析に置き換える。
    - **レート制限の高度化**: 短時間での過剰な接続試行自体をシステム全体で抑制するグローバル制限の導入。
