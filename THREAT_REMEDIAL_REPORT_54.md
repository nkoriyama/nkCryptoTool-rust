## 2. 実装詳細

### 2.1 診断性の向上 (F-53-1, F-53-2)
OpenSSL バックエンドが標準で PQC をサポートしていないことに起因する、不透明なエラー（"Failed to load..." 等）を排除しました。
- **OID スキャン**: 鍵データの DER バイト列をスキャンし、PQC 固有の OID または暗号化 PKCS#8 (PBES2) の OID を検出した場合、即座に「RustCrypto バックエンドの使用」を推奨する具体的なエラーを表示します。
- **フェイルファスト**: パスフレーズが提供されていない場合でも、データが PQC であることが判明した時点で早期にエラーを返し、不要な処理をスキップします。

### 2.2 リソース保護の最適化 (F-53-3)
ハンドシェイク（一時鍵生成等）のコストを攻撃者に消費させないため、防御レイヤーを二重化しました。
- **ID キャッシュ**: `IP_TO_PEER_ID` マップにより、特定の IP がどの PeerID（公開鍵ハッシュ）に紐付いているかを記憶します。
- **インテリジェント早期拒否**: 接続時に IP クールダウンだけでなく、キャッシュされた PeerID のクールダウン状態もチェックします。これにより、以前に正当性が確認されたピアが（攻撃目的で）再接続してきた場合、ハンドシェイクを試みる前に即座に切断することが可能になりました。

### 2.3 セキュリティ・デフォルトの維持
- `allow_unauth = false` のデフォルト設定と、認証済みピアに対する PeerID ベースの管理を組み合わせることで、NAT 共有環境での利便性と、悪意あるピアによる DoS 耐性を高い次元で両立させました。
