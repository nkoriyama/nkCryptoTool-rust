## 2. 実装詳細

### 2.1 検証体制の全面強化 (F-44-1)
F-43-1 で発生した「特定アルゴリズムのみのサイズ設定ミス」の再発を防ぐため、`utils::tests` において全PQCバリアントの鍵生成・包装・抽出・暗号/署名の往復検証を自動化しました。
- `test_pqc_kem_roundtrips`: ML-KEM-512, 768, 1024 を一括検証。
- `test_pqc_dsa_roundtrips`: ML-DSA-44, 65, 87 を一括検証。

### 2.2 API のクリーンアップ (F-44-3)
シード保存の廃止（F-43）に伴い、内部 API が「シードが返る可能性」を示唆し続けることによる誤解を排除しました。戻り値を `Result<Zeroizing<Vec<u8>>>` に縮小し、呼び出し側（PqcStrategy や NetworkProcessor）のコードも簡略化しました。

### 2.3 パース精度の向上 (F-44-2)
ヒューリスティック走査において、内部の `AlgorithmIdentifier` シーケンスなどを誤って秘密鍵本体と誤認する問題を修正しました。深さ 1 以上の再帰呼び出しでは、期待サイズにマッチしない場合に空の値を返すようにし、外側のループが正しい `OCTET STRING` を探し続けられるように改善しました。

## 3. 検証結果

- **単体テスト**: `cargo test --no-default-features --features backend-rustcrypto test_pqc` が全件 **PASS**。
- **結合テスト**: `test_pqc_e2e_cycle` および `test_pqc_interop_encryption_bidirectional` が正常に動作することを確認。
- **安全性**: `secure_write` によるファイル保護と `create_new(true)` による上書き防止機能が、テスト環境下でも正しく機能していることを実証しました。

## 4. 今後のロードマップ

- **ASN.1 構造的パースへの移行**: ヒューリスティックなバイト走査を卒業し、`pkcs8` クレート等を用いた完全な構造解析への移行を「中期目標」として維持します。
- **DoS 対策の強化**: チャットモードにおける IP ベースのレート制限や認証の義務化の検討。
